What is this website about ?

This website presents an analysis of land cover in Enschede, Netherlands. We classified an aerial ortho image of the city using two machine learning algorithms: Maximum Likelihood Classifier (MLC) and Support Vector Machine (SVM). We then assessed the accuracy of each classification and compared it to the existing official land cover product, LGN. This comparison allows us to evaluate the effectiveness of our classification methods and determine which algorithm provides the most accurate representation of Enschede's land cover.

We use Open Web Services (OWS), e.g. WMS and WFS, which embraces OGC standards. In this project, we only use WMS (Web Map Service) to provide a capability for requesting land cover classification map images (support PNG and JPEG format) by leveraging MapServer.

Workflow

Collect and explore data

Orthophoto

  • Description: The foundation of our land cover analysis for Enschede is a high-resolution orthophoto. This orthophoto was generated from digital airborne images captured by an UltraCamD, a large format camera known for its exceptional detail. The camera boasts a focal length of 105.2 millimeters and utilizes pixels measuring only 9 microns each.
  • Year: 2008
  • Extent: The minimum X coordinate is 256,583.20 meters, the minimum Y coordinate is 470,627.60 meters, the maximum X coordinate is 258,097.10 meters, and the maximum Y coordinate is 471,960 meters.

LGN

  • Description: The National Land Use File of the Netherlands (LGN) is a nationwide database and provides the user with up-to-date and accurate information about land use in the Netherlands. The LGN is created with the aim of providing ministries, provinces, water boards and other regional/national organisations with up-to-date information on land use and land use changes over time.
  • Year: 2018 (the closest year to orthophoto that we can find)
  • Spatial resolution: 5 meters
  • Land use classes: 51 classes (divided between agriculture, forest, water, nature, urban area)
  • Extent: By clipping the raster, we also get the same extent as orthophoto.

* The reason that we used LGN product to compare with classification product from orthophoto is we can't find classification product that has high spatial resolution as orthophoto and LGN is the highest spatial resolution classification product that we can find. Unfortunately, the LGN product is only available from year 2018.

Classification

To classify the land cover, we use supervised classification which are maximum likelihood (MLC) and support vector machine (SVM).

Maximum Likelihood Classification (MLC)

MLC has as assumption that the pixel values in each class is normally distributed. The pixel will be assigned to the class that has the highest probability value. From the above figure, the probability of x  belonging to class B is higher than class A.

Support Vector Machine (SVM)

SVM aims to find the optimal hyperplane to separate different classes and maximized the margin. If the data is non-linearly separable, we can use kernel functions to transform the data into higher-dimensional space. The example of of kernel functions are Linear, Polynomial, Sigmoid, and Radial Basis Function (RBF).

For our classification, we use Radial Basis Function (RBF) kernel Type.

Reference:

  • https://www.nv5geospatialsoftware.com/docs/SupportVectorMachine.html
  • https://www.redalyc.org/journal/2570/257059542002/html/

Uncertainty

Uncertainty in our product classification

  • Variation of pixel values in class and between classes: According to ROI separability report, some classes has low value of separability value such as Built-up area and Bare land. Moreover, standard deviation of Built-up area class in red, green, and blue band are high due to heterogeneous rooftops. These can lead to false classification.
  • Spectral resolution: We only have RGB band in the image and some objects like water, bare land, and vegetation are hard to distinguish compared to NIR band.
  • Shadow Effects: Shadows caused misclassification especially in SVM. However, we have tried to classify the shadows into one class to see the result as well. Click here to see more details
  • Relief displacement in the original image: There are still some relief displacement in the orthophoto that we used for classification. Relief displacement can make the classification generate mixed classes in one building.
  • Spatial sampling area: The accuracy is more rely on where we take the test sample, but we try to be unbias in collecting the test sample.
  • Number of training samples: Increasing training data can help improve accuracy, but it also increases the processing time especially in SVM.

Uncertainty in existing product's accuracy assessment

  • Reference Data Merging: We merged different classes in existing product based on our understanding such as classified "other rural areas” as "built-up" and "mudflats" merged into the "vegetation" class.
